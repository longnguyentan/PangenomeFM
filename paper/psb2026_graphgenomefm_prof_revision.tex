\begin{abstract}
Human pangenome graphs represent genetic variation more naturally than a
single linear reference, but they also require representation learning methods
that preserve graph topology rather than reducing the graph to one linear
path. We present GraphGenome-FM, a graph-native model pretrained with a
self-supervised edge-prediction objective on human pangenome graphs. The model
combines coordinate-aware attention, graph-topological attention, and learned
fusion to produce node embeddings for graph structure prediction and downstream
functional genomics tasks. GraphGenome-FM generalizes robustly across held-out
HPRC chromosomes and transfers to independently parsed HGSVC3 graphs without
retraining, supporting the portability of learned pangenome graph
representations. Initial ENCODE cCRE prediction experiments show that
regulatory annotation is a challenging but biologically meaningful downstream
task, motivating fairer comparison against linear-reference and
linearized-graph baselines as well as category-specific regulatory analysis.
Together, these results support graph-native pretraining as a foundation for
pangenome representation learning and motivate future applications in graph
enhancement and haplotype-specific functional annotation.
\end{abstract}

\keywords{Pangenome graph; graph neural network; self-supervised learning; HPRC; HGSVC; cCRE; link prediction.}

\section{Introduction}

The human reference genome is increasingly moving from a single linear
coordinate system toward graph-based references that encode common and rare
variation across multiple haplotypes. The Human Pangenome Reference Consortium
(HPRC) pangenome and related resources such as HGSVC provide graph
representations in which nodes are DNA segments and edges encode allowable
adjacencies across reference and alternate haplotype paths.\cite{liao2023hprc,ebert2021hgsvc,garrison2018vg}
These graphs preserve structural variation, allelic alternatives, and local
topological context that are difficult to express in a single reference
sequence.

Most genomic representation learning remains sequence-first. Linear models can
learn strong nucleotide-level representations, but they generally operate on
one path through the genome or on serialized graph traversals. This
serialization loses exactly the information that makes a pangenome graph
useful: branching, reconvergence, path multiplicity, and the relationship
between reference sequence and non-reference haplotypes. A graph-native
foundation model should learn from both coordinate continuity and graph
topology.

We study self-supervised link prediction as a pretraining objective for human
pangenome graphs. Given a local graph slice and candidate node pair, the model
predicts whether the edge is present in the pangenome graph. This task is
attractive because labels are intrinsic to the graph, the objective scales to
unlabeled pangenome data, and the resulting node embeddings can be reused for
downstream functional or variant annotation tasks.

This paper makes four contributions. First, we construct a corrected
all-chromosome HPRC benchmark with 24 GRCh38 chromosomes, 10 windows per
chromosome, and both strict and 1-hop graph closures. Second, we train a
shared dual-stream graph attention model across chromosomes and evaluate it on
chromosomes held out from training. Third, we audit the link-prediction task
with distance-matched negative edges and test external transfer to HGSVC
graphs. Fourth, we map ENCODE cCRE labels to HPRC graph nodes and use
regulatory element prediction as a downstream stress test. The strongest
current evidence is for graph-structure pretraining and cross-graph transfer;
the downstream cCRE experiments define the next methodological work.

We evaluate GraphGenome-FM in four stages. First, we pretrain and test on
chromosome-held-out HPRC graph slices. Second, we audit the task with
distance-matched negative edges to reduce concern that random non-edges make
the benchmark trivial. Third, we evaluate external HPRC-to-HGSVC3 transfer and
outline an HGSVC graph-imputation application. Fourth, we use ENCODE cCRE
prediction as a regulatory annotation stress test, with explicit attention to
baseline alignment, reduced cCRE label groupings, and biologically interpretable
category-specific performance.

\section{Related Work}

\subsection{Pangenome Graphs}

Variation graphs represent genomes as paths through shared sequence nodes and
alternate allelic branches. The variation graph toolkit demonstrated that this
representation can improve read mapping by representing genetic diversity
directly in the reference structure.\cite{garrison2018vg} The HPRC pangenome
extends this idea at population scale, providing a richer reference for human
genetic variation than GRCh38 alone.\cite{liao2023hprc} HGSVC resources
provide complementary haplotype-resolved assemblies and structural variation
catalogs that are useful for external graph-transfer evaluation.\cite{ebert2021hgsvc}

\subsection{Graph Neural Networks}

Graph neural networks aggregate information over edges and have become a
general tool for semi-supervised and self-supervised learning on relational
data.\cite{kipf2017gcn,velickovic2018gat} Graph Attention Networks learn
edge-dependent attention weights, making them a natural starting point for
pangenome graphs where local topology, branching, and path alternatives can
change substantially across genomic loci. Our model builds on this family but
adds an explicit coordinate-aware stream because pangenome nodes also have
genomic offsets and orientations.

\subsection{Genomic Foundation Models and Linearized Graph Models}

Recent genomic foundation models have shown that self-supervised learning can
capture useful biological signals from DNA sequence.\cite{ji2021dnabert}
Pangenome graphs raise a distinct representation problem: the input object is
not only a string but a structured graph with multiple valid haplotype paths.
For cCRE and other downstream tasks, we will compare graph-native modeling to
linear-reference models and, where feasible, linearized-pangenome or
linearized-graph models such as DeepGene-style representations.

\section{Methods}

\subsection{HPRC and HGSVC Graph Processing}

We use cleaned pangenome graph tables derived from rGFA/GFA input. Segment
tables include node identifiers, sequence length, chromosome or sequence name,
offset, and sample or reference status. Link tables define directed graph
edges with endpoint orientation. HPRC is the primary training and internal
evaluation graph. HGSVC3 is the primary independent external graph for
transfer evaluation, using the currently parsed CHM13 targets chr19, chr21,
chr22, and chrY. ENCODE cCRE annotations are mapped to HPRC GRCh38 nodes for
downstream functional annotation.

Although additional public pangenome resources are becoming available, their
graph representations differ in structure and path annotation. We therefore
focus the current study on HPRC and HGSVC, which use more comparable graph
construction workflows in the current parsed resources, and leave broader
multi-resource harmonization for future work.

\subsection{Graph Slice Construction}

For each HPRC target chromosome, we sample 10 windows of 50 kb and construct
two closures. The strict closure contains the target reference path nodes and
edges within the window. The 1-hop closure expands the local graph by adding
direct graph neighbors, capturing nearby alternate haplotype structure. The
corrected HPRC benchmark therefore contains
\[
24 \; \mathrm{chromosomes} \times 10 \; \mathrm{windows} \times
2 \; \mathrm{closures} = 480
\]
graph slices. HGSVC3 transfer slices use the same strict and 1-hop closure
definitions on the parsed CHM13 targets.

\subsection{Negative Edge Sampling}

Each link-prediction slice contains positive graph edges and an equal number
of negative candidate edges. The original HPRC benchmark used random
non-edges. Because random negatives can be too easy if they differ greatly in
genomic distance or graph context from real edges, we implemented a
distance-matched negative sampler. For a positive edge, the sampler proposes
non-edge endpoint pairs from the same sequence context whose coordinate
separation is matched within a tolerance of 1 kb or 10 percent of the
positive edge distance. This rerun is designed as a paper-safety audit.

\subsection{GraphGenome-FM Architecture}

GraphGenome-FM uses a shared DualStreamPangenomeGAT backbone. Each graph
slice is converted to oriented nodes so that segment orientation can influence
attention and prediction. Raw node features include sequence length, genomic
offset, reference or sample status, graph degree, local branching fraction,
and orientation-derived features. The model first maps these features into a
hidden representation and then applies two streams:

\begin{itemlist}
\item A coordinate stream applies self-attention with multi-scale rotary
position embeddings over genomic offsets.
\item A graph stream applies GAT-style message passing over observed graph
edges to capture local topology, branching, and reconvergence.
\item A learned fusion gate combines the coordinate and graph streams on a
per-node basis before edge prediction.
\end{itemlist}

The experiments reported here use hidden dimension 48, 4 attention heads, 2
layers, focal binary cross-entropy with $\gamma=2.0$, DropEdge regularization
with rate 0.1, a 5-epoch warmup, and cosine learning-rate decay. The edge
predictor is an MLP over concatenated endpoint embeddings.

\subsection{Training and Chromosome-Held-Out Evaluation}

The HPRC shared pretraining split is chromosome-held-out. Training uses all
canonical chromosomes except chr1, chr8, chr19, chrY, and validation chr16.
Chromosomes chr1, chr8, chr19, and chrY are held out for final testing.
Strict and 1-hop closures are trained and evaluated separately.

\subsection{External HGSVC Transfer}

For external transfer, we freeze the HPRC-trained checkpoint and evaluate it
directly on HGSVC3 graph slices without retraining. The original transfer
experiment used CHM13 chr22 only. The expanded experiment uses distance-matched
negatives and four parsed CHM13 targets: chr19, chr21, chr22, and chrY.

\subsection{HGSVC Graph Imputation Experiment}

The HGSVC graph-imputation experiment is a planned application pilot. The
design is to score candidate missing or weakly supported graph adjacencies in
older HGSVC graphs using the frozen HPRC-trained model, then compare the
highest-scoring candidate edges against a newer pangenome construction for the
same sample or region when available. Primary validation metrics will include
candidate-edge precision/recall, strict-closure recovery, 1-hop recovery, and
overlap with edges introduced by the newer graph build. No imputation result is
claimed in this draft until that validation is run.

\subsection{ENCODE cCRE Mapping}

GRCh38 graph nodes by interval overlap. The current mapped label table

\subsection{cCRE Classification Baselines}

We evaluate binary cCRE/non-cCRE prediction, reduced-group cCRE prediction, and
the original 9-class cCRE task. Logistic-regression baselines use all labeled
GRCh38 nodes and hand-crafted graph/coordinate features. Planned baselines
include an MLP over node features, an MLP over frozen GraphGenome-FM
embeddings, a linear-reference-only model, and a DeepGene-style linearized
graph model if a runnable implementation can be identified. GAT-based cCRE
models currently use only labeled nodes that appear in benchmark windows. This
mismatch is important: the current downstream GAT numbers should not yet be
treated as a fair all-node comparison against logistic baselines.

\subsection{Statistical Analysis and Metrics}

For link prediction and binary cCRE prediction, we report AUROC, AUPRC, Brier
score where available, macro-F1, and balanced accuracy. For multiclass cCRE
prediction, we emphasize macro-F1 because the cCRE labels are strongly
imbalanced. For cCRE comparisons, the next manuscript version will explicitly
report whether each model is evaluated on all labeled nodes or on the
benchmark-window subset.

\subsection{Downstream Functional Annotation Through cCRE Prediction}

\section{Discussion}

The central result of this draft is that a graph-native model can learn
pangenome edge structure across many HPRC chromosomes and transfer to HGSVC
without retraining. The distance-matched negative audit strengthens the claim
by showing that the link-prediction result is not explained only by easy
random negatives. The HGSVC transfer experiment provides an external
replication setting based on a related but independent pangenome graph source.

The cCRE downstream story is deliberately more cautious. The binary
logistic-regression baseline is already strong, whereas current binary and
multiclass GAT variants underperform. This does not invalidate the
pretraining result, because the GAT cCRE setting is not yet matched to the
baseline evaluation population; the current benchmark windows cover only 4.21
percent of the all-node logistic test universe. The next downstream
experiments should prioritize evaluation alignment, class imbalance,
category-aware analysis, and comparison against linear-reference and
linearized-graph models.

\subsection{Future Application: Haplotype-Specific Functional Annotation}

A longer-term application is to use graph-native representations to map
functional genomics signals directly onto haplotype paths, rather than first
projecting them onto a single linear reference. Current cCRE labels are derived
largely from functional genomics data aligned to a linear reference and then
summarized across samples or assays. A graph foundation model could eventually
help assign ATAC/DNase, methylation, histone-modification, CTCF, or 3D genome
signals to specific HPRC or HGSVC graph paths. This would support
haplotype-specific regulatory annotation, allele-specific functional analysis,
and better interpretation of functional elements in structurally variable
regions. We frame this as future work rather than a completed result.

\section{Limitations and Ongoing Work}

Several limitations remain before submission. The link-prediction task uses
synthetic negative edges; distance matching is a stronger control than random
sampling, but additional local hard-negative and degree-matched schemes should
be evaluated. HGSVC external transfer currently covers selected chromosomes,
not all possible external graph contexts. The cCRE GAT comparison is not yet
fair because GAT and logistic models evaluate different labeled-node
populations. Finally, final reproducibility experiments should be rerun on a
CUDA-enabled lab or cluster environment with fixed software versions and saved
logs.

Although additional public pangenome resources are becoming available, their
graph representations differ in structure and path annotation. We therefore
focus the current study on HPRC and HGSVC, which use more comparable graph
construction workflows, and leave broader multi-resource harmonization,
including 1000 Chinese Pangenome-style resources, for future work.

Immediate next steps are: (i) align cCRE GAT and logistic evaluation by using
the same labeled-node universe; (ii) build reduced-category cCRE tasks; (iii)
obtain richer ENCODE/SCREEN metadata if signal or Z-score stratification is
desired; (iv) add linear-reference and DeepGene-style comparisons; (v) promote
the draft figures into final panels; and (vi) pilot the HGSVC graph
imputation/enhancement application if time allows.

\section{Conclusion}

GraphGenome-FM provides preliminary evidence that self-supervised graph-native
pretraining can learn transferable representations of human pangenome graph
structure. HPRC chromosome-held-out results and HGSVC external transfer form
the strongest current story. The cCRE experiments identify a harder
downstream challenge: converting graph pretraining into regulatory annotation
gains under fair, biologically meaningful evaluation. The next paper revision
will strengthen this downstream section with aligned evaluation, external
comparative methods, cCRE category analysis, and application-oriented pilots.
